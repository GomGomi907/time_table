# 문제 분석 보고서: 구글 캘린더 연동 이슈

## 1. "설정 페이지에 들어가지지 않음"
- **현상**: 위젯 상단에 있는 톱니바퀴(설정) 아이콘을 눌러도 반응이 없습니다.
- **원인**:
    - 현재 디자인 상의 `톱니바퀴 아이콘`은 시각적인 요소(Mock UI)로 배치되었으며, 실제 클릭 이벤트나 기능이 구현되어 있지 않습니다.
    - 구글 연동 기능은 별도의 **"Calendar Sync" 위젯** 내 "Connect Google" 버튼을 통해 제공되고 있습니다.
- **해결 방안**:
    - 당분간은 위젯 내부의 버튼을 이용해주시기 바랍니다.
    - 추후 설정 모달(Modal) 창을 구현하여 톱니바퀴와 연결할 예정입니다.

## 2. "실제 데이터가 보이지 않음"
- **현상**: 구글 로그인을 완료하고 "Connected" 상태가 되었음에도, 중앙의 타임테이블 위젯에는 구글 일정이 표시되지 않습니다.
- **원인**:
    - **데이터 수집 성공**: 백엔드 서버는 구글 API로부터 일정을 정상적으로 가져와 데이터베이스(`google_events` 테이블)에 저장하고 있습니다. (기능 정상)
    - **데이터 전달 누락**: 프론트엔드가 데이터를 요청하는 API (`GET /api/mvp/routine`)가 현재 **고정된 가짜 데이터(Rule-based Routine)**만 반환하도록 설계되어 있습니다. 데이터베이스에 저장된 구글 일정을 조회해서 합치는 로직(**Event Merging**)이 아직 적용되지 않았습니다.
- **해결 방안 (Event Merging)**:
    - 백엔드 API를 수정하여, 데이터를 반환할 때 **[고정 루틴] + [DB의 구글 일정]**을 합쳐서 보내도록 변경해야 합니다.
    - 이 작업이 완료되면 프론트엔드 수정 없이도 일정이 보이게 됩니다.

2.  **검증**: 타임테이블에 "회의", "점심 약속" 등 구글 일정이 섞여서 나오는지 확인합니다.
