# Time Table final presentation demo script

## 10–15분 발표 구성

1. 0:00–1:30 문제 정의
   - 일정, todo, 목표, 집중 맥락이 흩어져 있으면 실행으로 들어가기 어렵다.
   - 개인적 동기는 ADHD 관련 실행 기능 부담을 줄이는 도구를 직접 설계해 보는 것이다.

2. 1:30–3:00 해결 목표
   - Time Table은 캘린더가 아니라 실행형 워크스페이스다.
   - 오늘 브리핑, 주간 시간표, 목표, 집중 모드, AI 제안을 한 흐름으로 묶는다.

3. 3:00–8:00 화면/구현 설명
   - 로그인/온보딩: 개인 생활 리듬 입력
   - 대시보드: 오늘 일정과 다음 행동 요약
   - 주간 일정: mock data로 발표 준비 흐름 표시
   - 집중 모드: 지금 해야 할 작업을 크게 표시

4. 8:00–11:00 짧은 live demo
   - 배포 링크 또는 로컬 실행 화면을 연다.
   - 로그인 → 대시보드 → 주간 일정 → 일정 추가 → 집중 모드를 순서대로 클릭한다.
   - 링크만 던지지 않고 직접 흐름을 보여준다.

5. 11:00–13:00 현재 상태와 남은 작업
   - MVP demo-ready 범위와 Google/AI/배포 안정화 과제를 구분한다.

6. 13:00–15:00 마무리
   - 문제해결프로그래밍 관점에서 문제 정의, 설계, 구현, 검증의 흐름을 요약한다.
