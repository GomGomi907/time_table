# 📝 프로젝트: Personal Holiday Dashboard (v2.0)

## 1. 개요

* **목표:** 사전에 계획된 타임테이블을 기반으로 현재 할 일을 직관적으로 보여주고, 사용자의 실제 수행 여부(완료/보류)에 따라 상태를 관리하는 웹 애플리케이션.
* **핵심 가치:** 계획 준수율 향상 및 "지금 무엇을 해야 하는가"에 대한 즉각적인 인지.

## 2. 주요 기능 명세

### A. 타임테이블 입력 및 파싱 (Input System)

* **텍스트 입력 모드:**
* 사용자가 텍스트 에리어에 일정 텍스트를 붙여넣기 하면 자동으로 파싱하여 일정 데이터로 변환.
* *지원 포맷:* `HH:mm-HH:mm [내용]` (예: `14:50-15:20 빨래 마무리`)
* *자동 날짜 인식:* 입력된 텍스트의 날짜(예: `2/16(월)`)를 인식하여 해당 날짜의 스케줄로 분류.


* **데이터 저장:**
* `localStorage`를 활용하여 새로고침 해도 데이터가 유지되도록 구현.



### B. 메인 대시보드 (Dashboard View)

* **현재 상태 (Current Focus):**
* 가장 크게 표시.
* **표시 정보:** 현재 작업명, 시작-종료 시간, 남은 시간(분/초), 진행률(Progress Bar).
* **유휴 상태:** 일정 사이의 빈 시간이나 '완료' 후 남은 시간은 "자유 시간(Free Time)" 또는 "대기 중"으로 표시.


* **다음 일정 (Next Up):**
* 하단에 작게 표시.
* 바로 다음에 올 일정의 시간과 제목 표시.



### C. 인터랙티브 제어 (Control Features)

* **✅ 완료 (Complete):**
* 현재 진행 중인 일정을 즉시 '완료' 처리.
* *동작:* 남은 시간이 있어도 진행률을 100%로 채우고, 다음 일정 시작 시간 전까지 "자유 시간" 모드로 전환.


* **⏸ 보류 (Hold/Pause):**
* 작업 중단이 필요할 때 사용.
* *동작:* 타이머/진행률 바 정지. 상태 메시지를 "일시 정지됨"으로 변경. 배경색 변경(예: 노란색/회색)으로 시각적 알림.
* *재개:* '재개(Resume)' 버튼을 누르면 다시 타이머 작동.



### D. 데이터 시각화 및 알림

* **진행률 바 (Progress Bar):**
* 시간 경과에 따라 자동으로 차오름 (CSS Animation/JS).


* **종료 임박 알림:**
* 일정 종료 5분 전, 1분 전에 시각적 알림(텍스트 색상 변경 또는 깜빡임).

## 3. 화면 구성안 (UI Draft)

| 영역 | 구성 요소 | 비고 |
| --- | --- | --- |
| **Header** | 현재 날짜/시간, [일정 수정] 버튼 | 우측 상단 설정 아이콘 |
| **Main** | **[현재 작업명]** (초대형 텍스트) |  |
|  | 진행률 바 & 남은 시간 타이머 |  |
|  | **[완료]** **[보류]** 버튼 그룹 | 큼직한 터치 타겟 |
| **Footer** | Next: [시간] [다음 작업명] | 반투명 처리 |
| **Modal** | 텍스트 입력창 (타임테이블 붙여넣기용) | 설정 버튼 클릭 시 노출 |
