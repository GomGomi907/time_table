# Time Table Screen Definition

이 문서는 화면이 다시 장식 위주로 흔들리지 않도록, 각 페이지가 무엇을 보여줘야 하는지 먼저 고정하는 기준이다.

## 핵심 원칙

1. 첫 화면은 "지금 무엇을 해야 하는가"를 바로 보여준다.
2. 한 화면에 한 역할만 둔다. 셸은 이동과 상태, 본문은 작업만 담당한다.
3. 장식 문구보다 실제 데이터가 먼저 보여야 한다.
4. 영어 메타포와 SF 연출은 쓰지 않는다.
5. 모바일은 정보량을 줄이는 대신 조작 흐름을 더 단순하게 만든다.

## 전역 셸

- 사이드바와 모바일 상단 셸은 다음만 담당한다.
- 현재 위치 확인
- 전역 이동
- 로그인 계정 / Google 연동 상태
- 로그아웃

금지 사항:

- 페이지 소개 문구 반복
- 히어로 카피
- 제품 철학 설명

## 로그인 화면

목적:

- 사용자가 "로그인하면 무엇을 얻게 되는지"를 짧게 이해하고 바로 진입하게 한다.

필수 요소:

- 제품명
- 한 줄 가치 제안
- Google 로그인 CTA
- 로그인 후 보게 될 핵심 화면 3가지 요약

금지 사항:

- 과도한 상하 여백
- 의미 없는 번호 라벨
- 마케팅용 장식 박스

## 대시보드

목적:

- 오늘 움직이기 위한 정보만 압축해서 보여준다.

필수 블록:

- 현재 일정
- 다음 일정
- 주요 할 일
- 오늘 일정 목록
- 지금 챙길 목표

규칙:

- 첫 카드에서 오늘의 목적을 설명하고 끝낸다.
- 나머지는 모두 실제 데이터 카드로 구성한다.
- 빈 상태는 "데이터 없음"이 아니라 다음 행동을 안내한다.

## 시간표

목적:

- 주간 블록을 읽고 수정하고 가져오는 작업을 한 화면에서 끝낸다.

필수 블록:

- 이번 주 요약 지표
- 요일별 일정 카드
- 텍스트 기반 일정 가져오기
- 상태 메시지
- 블록 생성 / 수정 모달

모바일 원칙:

- 7열 그리드를 고집하지 않는다.
- 하루 단위 카드 흐름을 유지한다.

## 목표

목적:

- 목표를 상태별로 나누어 빠르게 정리하고 수정한다.

필수 블록:

- 목표 요약 지표
- 대기 / 진행 / 완료 / 위험 4개 컬럼
- 생성과 수정을 모두 처리하는 공용 드로어

금지 사항:

- 플레이스홀더 카드
- 보드 아래 중복 입력 폼
- 완료와 위험을 한 컬럼에 섞는 구조

## 설정

목적:

- 엔진 파라미터와 연동 상태를 한 화면에서 확인하고 바로 저장한다.

필수 블록:

- 현재 연동 상태
- 주요 시간 규칙 요약
- 엔진 파라미터 폼
- 저장 결과 메시지

규칙:

- 백엔드 필드와 UI 필드는 1:1로 대응한다.
- 저장 결과는 검은 박스가 아니라 표준 상태 메시지로 노출한다.

## 시각 기준

- 배경은 따뜻한 중성 톤을 유지한다.
- 카드는 밝고 읽기 쉬워야 한다.
- 강조색은 액션과 진행률에만 사용한다.
- 제목은 짧고 강하게, 본문은 설명보다 안내에 가깝게 쓴다.
- 한 화면에서 가장 먼저 보여야 하는 것은 항상 버튼이 아니라 데이터다.
